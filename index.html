Projet Principal

Projet phare

Pipeline d'analyse de satisfaction client

ShowroomPrivé : du scraping au dashboard décisionnel

Contexte et problématique

ShowroomPrivé accumule des milliers d'avis clients sur Trustpilot, impossible à exploiter manuellement à cette échelle. L'objectif : automatiser leur collecte, détecter automatiquement les sentiments négatifs et fournir aux équipes métier un tableau de bord décisionnel actualisé en continu. Du scraping à l'alerte Prometheus, chaque étape du pipeline est orchestrée, testée et monitorée.

Architecture et pipeline ETL

Pipeline de bout en bout : MongoDB → Airflow → MySQL → FastAPI → Prometheus / Grafana

  • Ingestion : scraping de 20 259 avis via BeautifulSoup, migration des données brutes de MongoDB vers MySQL orchestrée par Airflow (98,7 % d'intégrité).
  • Modélisation : schéma relationnel optimisé, scripts de transformation et tests de qualité des données.
  • Machine learning : analyse de sentiments avec TF-IDF, régression logistique, Random Forest, SVM et CamemBERT. Réentraînement automatique, F1-score de 0,92.
  • API REST : FastAPI avec documentation Swagger, endpoints /predict, /stats et health check, latence inférieure à 200 ms.
  • Orchestration : DAG Airflow à 7 tâches (scraping, nettoyage, agrégation, stockage, reporting, alertes, ML).
  • Infrastructure : Docker Compose multi-services (MySQL, MongoDB, Airflow, API, monitoring).
  • Observabilité : stack Prometheus, Grafana et StatsD pour métriques temps réel et alerting automatique.

Qualité et DevOps

  • Architecture modulaire : src/ingestion, src/transform, src/utils. Chaque module est indépendant et testable.
  • Tests et formatage : Black, Flake8, pytest avec 87 % de couverture.
  • CI/CD : GitHub Actions pour build, tests et déploiement en staging.
  • Sauvegarde : scripts automatisés mysqldump et mongodump.
  • Documentation : README technique complet, ERD, diagrammes d'architecture, OpenAPI/Swagger.
