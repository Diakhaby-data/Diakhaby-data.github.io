À propos

Data Engineer : transformer vos données en valeur stratégique

Data Engineer, j'aide les entreprises à transformer leurs données brutes en informations stratégiques grâce à des pipelines robustes, automatisés et scalables. Formé chez DataScientest, j'ai développé une expertise pratique en ingestion multi-sources, modélisation SQL/NoSQL et orchestration avec Airflow et DBT.

Projet Phare

⭐ PROJET PRINCIPAL

🎯 Pipeline d'Analyse de Satisfaction Client

ShowroomPrivé · Projet de formation DataScientest

🎯 Contexte & Challenge

Concevoir un pipeline complet d'analyse de satisfaction client e-commerce, de la collecte web à la prédiction ML, avec exposition API et dashboard décisionnel temps réel. Objectif : analyser la satisfaction client pour détecter les points d'amélioration du service après-vente de ShowroomPrivé.

🗺️ Architecture Complète & Pipeline ETL

Pipeline end-to-end : MongoDB → Airflow → MySQL → FastAPI → Prometheus/Grafana

  • Ingestion automatisée : Scraping 20 259 avis (BeautifulSoup) + migration des données brutes depuis MongoDB vers MySQL avec orchestration Airflow (98,7% d'intégrité des données)
  • Modélisation relationnelle : Schéma SQL optimisé avec scripts de transformation et tests de qualité des données
  • Machine Learning : Pipeline d'analyse de sentiments (TF-IDF + Régression Logistique, RandomForest, SVM, CamemBERT) avec réentraînement automatique - F1-score = 0.92
  • API REST production-ready : FastAPI avec documentation Swagger (endpoints /predict, /stats, health check) - latence < 200 ms
  • Orchestration Airflow : DAG automatisé 7 tâches (scraping → cleaning → aggregation → MongoDB → reporting → alertes → ML)
  • Containerisation complète : Architecture Docker multi-services (MySQL, MongoDB, Airflow, API, monitoring)
  • Observabilité : Stack complète Prometheus + Grafana + StatsD pour métriques temps réel et alerting automatique

🔧 Qualité & DevOps

  • Code structuré : Architecture modulaire Python (src/ingestion, src/transform, src/utils)
  • Tests & Qualité : Black (formatage), Flake8 (linting), Pytest (tests unitaires, 87% couverture)
  • CI/CD : GitHub Actions (build, tests, staging automatisés)
  • Sauvegarde & Restauration : Scripts automatisés MySQL (mysqldump) et MongoDB (mongodump)
  • Documentation : README technique complet, ERD, architecture diagrams, OpenAPI/Swagger
